隨著行動科技逐漸普及，觀光客可以輕易從網路上獲取旅遊資訊。在現在的數位時代，紙本的觀光地圖、旅遊書等，已逐漸由智慧型手機取代。「森遊阿里山」將日出時間、交通資訊、園區地圖等多樣化的資訊統合在同一款應用程式，觀光客可以不必擔心找不到地圖，為了交通資訊苦惱，縮短找資訊的時間，更自在享受旅行。另外，在軟體產品開發過程，「易用性」普遍是在團隊中較少被重視的環節。易用性評估測試不僅可以在軟體正式開發前避免嚴重的易用性問題，開發後定期做易用性評估測試可用來來維護及調整產品。 本研究將使用易用性評估測試研究方法，第一步驟為自我介紹及任務測試事前說明；第二步驟為受測者任務操作，同時紀錄及觀察；第三步驟為使用後進一步訪談，錄影並紀錄受測者的使用動機、使用直覺、使用偏好習慣；第四步驟為受測者的後續回饋如整體產品印象、使用改善建議、產品操作滿意度。

Data record
此次易用性測試將記錄以下數據：

  • 受測者任務花費時間。
  • 受測者完成進度。
  • 受測者操作任務時點擊螢幕的次數。
  • 受測者操作任務點擊失敗次數。
  • 受測者對於此任務使用上的滿意程度。
Evaluation index
  • 任務成功(Task Success)：受測者是否有完成任務。
  • 任務錯誤(Task Failures)：受測者執行任務時的非預期錯誤，如錯過、按錯按 鈕，導致任務失敗。在處理錯誤時，必須了解導致錯誤的前因後果，以利改善。
  • 任務時間(Workflow Time)：操作任務時所花費的時間。
  • 滿意度(Task Rating)：受測者在操作任務時的滿意程度對於這次任務操作的評 價。此項指標最後將使用五位受測者的分數以SUS量表計算出產品滿意度分數。
Executive
Brief
主持人在受測者操作任務之前，須將下列指示明確地告訴受測者：
您好，我們將進行五項「森遊阿里山」APP易用性任務測試。我們將會詢問您的個人基本資料，並在您操作每項任務時錄影紀錄。另外在每項任務完成時，會詢問您10題關於操作任務時的滿意度，並期望能收到您在操作任務時的其他心得。 此次測試的目的是評估「森遊阿里山」APP的易用性，找出使用性問題及改善的地方。您不需要有使用過此APP的經驗或相關的知識，我們是在測試產品的易用性，並非測試您，請放輕鬆操作所有任務，謝謝您。

  • 任務時間(Workflow Time)：操作任務時所花費的時間。

Result analysis and Conclusion
本工作坊透過對導覽型APP的易用性操作任務測試，了解介面在引導使用者操作、資訊架構設計上，是否對使用者友善，並依結果提供修改建議。
Reflection
During this project, I encountered several issues while communicating with the engineers. I noted them down as reminders for improvement in the future!
  • Missing Font Characters
    When using signature files, I designed three fonts for users. One handwritten font was pointed out by the engineer for missing characters, so I replaced it with another one.
  • Understanding File Formats
    The IMG file mentioned in the project was misunderstood as a new kind of scanned image. Later, the engineer explained that IMG is just a general term for formats like JPG and PNG.
  • Accuracy in Wording
    After getting feedback, I reviewed the project and found that I mistakenly wrote "MB" as "Mb" in the file instructions. After researching, I confirmed that MB and Mb are different units.
